Day-30 | 贪心算法数组重叠问题

Posted on 2025-02-18 Edited on 2025-04-16

今天算是贪心算法最简单的一天了，可以自己相处解法还是很有成就感

  1. Arrows to burst ballons

  2. 按气球的右边界升序排序。

  3. 初始化箭数为 1（至少一支箭），设 end 为第一个气球的右边界。

  4. 遍历后续气球，如果当前气球的左边界大于 end，说明它与前面的组不重叠，需要新开一支箭，同时更新 end 为当前气球的右边界。

  5. Non-overlapping Intervals

  6. 按intervals的右边界升序排序。

  7. 初始化toRemove为0（没有需要removed的），设 end 为第一个数组的右边界。

  8. 遍历后续intervals，如果当前interval的左边界小于 end，说明它与前面的组重叠，需要toRemove++，并continue跳过此interval；相反，如果左边界大于或等于end，更新 end 为当前气球的右边界。

  9. Partition Labels
    end：当前片段中所有字符的最后出现位置的最大值。
    常用的更新最大位置的写法

    当遍历到的索引 i 等于 end 时，说明从 start 到 end 范围内的所有字符，其最后出现位置都在该区间内，当前片段就可以确定。
    当确定一个片段（即 i == end）时，将该片段的长度 end - start + 1 加入结果列表，并将 start 更新为 i + 1 开始下一个片段的划分。
