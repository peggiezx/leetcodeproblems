代码随想录训练营打卡 Day-32 ｜ 动态规划基础题01

Posted on 2025-02-23 Edited on 2025-04-01 In dp

动态规划的五个步骤每一步都不能能省:

  1. 确定dp数组（dp table）以及下标的含义

  2. 确定递推公式

  3. dp数组如何初始化

  4. 确定遍历顺序

  5. 举例推导dp数组

  6. 确定 dp 数组及下标含义
    爬楼梯问题
    定义：令 dp[i] 表示到达第 i 阶楼梯的方法总数。

最小花费爬楼梯问题
定义：令 dp[i] 表示到达第 i 阶楼梯时所花费的最小总花费。
（注意：题目通常规定可以从 0 或 1 阶开始，因此初始化时需考虑这一点。）

  1. 确定递推公式
    爬楼梯问题
    每一步只能跨 1 阶或 2 阶，因此到达第 i 阶可以由以下两种情况得到：

从第 i-1 阶走一步上来
从第 i-2 阶走两步上来
所以递推公式为：
dp[i] = dp[i-1] + dp[i-2]
最小花费爬楼梯问题
每一步可以选择跨 1 阶或 2 阶，并且每走到一个台阶需要付出对应的花费 cost[i]。
那么到达第 i 阶的最小花费为：
dp[i] = cost[i] + min(dp[i-1], dp[i-2])

  1. dp 数组如何初始化
    爬楼梯问题
    常见的初始化方式：

dp[0] = 1 （假设站在第 0 阶，即楼梯底部，有一种“原始状态”）
dp[1] = 1 （只有一种方法从 0 到 1 阶）
（有的变种会用 dp[1] 和 dp[2] 来初始化，这取决于题目如何描述起点）

最小花费爬楼梯问题
由于可以从 0 或 1 阶开始，所以：

dp[0] = cost[0]
dp[1] = cost[1]
4. 确定遍历顺序
爬楼梯问题
递推公式依赖于 dp[i-1] 和 dp[i-2]，因此遍历顺序应从小到大：
从 i = 2 开始一直推导到目标楼梯 n。

最小花费爬楼梯问题
同样，由于 dp[i] 依赖于前两个状态，所以遍历顺序也是从 i = 2 开始，直到最后一个台阶。

  1. 举例推导 dp 数组
    爬楼梯问题示例（假设 n = 3）：

初始化：dp[0] = 1, dp[1] = 1
dp[2] = dp[1] + dp[0] = 1 + 1 = 2
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
表示共有 3 种方法到达第 3 阶楼梯。
最小花费爬楼梯问题示例（假设 cost = [10, 15, 20]）：

初始化：dp[0] = 10, dp[1] = 15
dp[2] = cost[2] + min(dp[1], dp[0]) = 20 + min(15, 10) = 20 + 10 = 30
最后，由于可以从倒数第一或倒数第二阶走到终点（相当于楼梯顶部），最终答案为 min(dp[1], dp[2]) = min(15, 30) = 15
表示最小花费为 15。
