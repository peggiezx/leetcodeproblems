代码随想录训练营打卡 Day-27 ｜ 贪心算法

Posted on 2025-02-13 Edited on 2025-02-14 In Greedy Algorithm

贪心算法的精髓在于寻找局部的最有解，如果没有反例则可以推倒全局最优解

  1. Assign Cookies

  2. Wiggle Sequence
    局部最优：
    在遍历过程中，每当当前数字与前一个数字之间的差值方向（正或负）与前一次的差值方向不同，就认为找到了一个“转折点”，可以把这个数字加入解中。
    全局最优：
    贪心地捕捉所有的转折点，便可以构造出最长的摆动子序列，因为只要每个转折点局部最优，整体也是最优解

    Alert 1: 当 diff == 0 时，不算作转折，因此不会更新 count 和 prevDiff。
    Alert 2: 初始状态的处理很关键，确保第一个非零差值能被正确捕捉。

  3. Maximum Subarray
    When sum is negative:
    We set sum to the current number because starting a new subarray from nums[i] is more beneficial than continuing with a negative sum.

When sum is non-negative:
We add the current number to sum, continuing the current subarray.

Updating max:
After each iteration, we update max with the current sum, ensuring that even if we reset sum in the if branch, the potential new maximum is captured.
