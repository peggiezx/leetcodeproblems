代码随想录训练营打卡 Day-35 | 动态规划背包问题03

Posted on 2025-02-24 Edited on 2025-04-01 In dp , knapsack

可以用回溯解决但是会超时，于是用动态规划解题

  1. Partition

  2. 确定 dp 数组以及下标含义
    状态定义：
    定义 dp[i][j] 表示：在前 i 个数中，是否存在一个子集，其和恰好为 j。
    其中 i 的取值范围是 0 ~ n（n 为数组长度），j 的取值范围是 0 ~ target（target = sum/2）。

  3. 确定递推公式
    对于每个数字 nums[i-1]，有两种选择：不选或选。因此状态转移可以写成：

不选当前数字： 如果 dp[i-1][j] 为 true，那么 dp[i][j] 也为 true。
选当前数字： 如果 j ≥ nums[i-1] 且 dp[i-1][j - nums[i-1]] 为 true，那么 dp[i][j] 也为 true。
综合起来有递推公式：
dp[i][j] = dp[i-1][j] || (j ≥ nums[i-1] && dp[i-1][j - nums[i-1]])

  1. dp 数组如何初始化
    初始状态：
    dp[0][0] = true，因为不选任何数字可以构造出和为 0。
    dp[0][j] = false，对于 j > 0，因为没有数字时不可能构造出正数和。
    对于任意 i，dp[i][0] = true，因为和为 0 总是可以通过不选取任何数字获得。

  2. 确定遍历顺序
    外层循环： i 从 1 到 n，表示逐步考虑前 i 个数字。
    内层循环： j 从 1 到 target，更新 dp[i][j] 的值。
    这种顺序确保每个 dp[i][j] 都是基于前 i-1 个数字的信息更新的。

  3. 举例推导 dp 数组
    以数组 [1, 5, 11, 5] 为例，总和 sum = 22，target = 11。
    构造 dp 数组时，初始状态为 dp[0][0] = true，其余 dp[0][j]（j > 0）为 false。

当 i = 1（只考虑数字 1）时：
dp[1][0] = true
dp[1][1] = true（因为 dp[0][1-1]=dp[0][0] 为 true）
其他 j 值保持 false。
当 i = 2（考虑数字 1 和 5）时：
保持上一行的状态，同时更新 dp[2][5] = true（由 dp[1][0] 得到），dp[2][6] = true（由 dp[1][1] 得到），依此类推。
最终，如果 dp[4][11] 为 true，则说明可以找到子集和为 11。
