代码随想录训练营打卡 Day 15 ｜513, 112, 113, 105, 106

Posted on 2025-01-19 Edited on 2025-01-27 In Medium

二叉树的重建从naive approach到optimized version，只要头脑清晰就可以把boundary的指数做对，慢慢来！

  1. Find Bottom Left Tree Value
  • use bfs, scan level by level
  • SMART PART: scan it from right to left, to make sure the last node is the leftmost node
  1. Path Sum (判断是否存在和为targetSum的path)
  • 用做减法的方法让targetSum - root.val
  • 终止条件：
    • root == null
    • 判断是否存在：如果是leaf node 并且targetSum==root.val
  • 运算逻辑：
    • target -= target - root.val
    • 用新的target检测左子树和右子树
  1. Path Sum II（记录所有和为targetSum的path）

  • 需要backtrack辅助method

  • 终止条件：

    • root==null

  • 运算逻辑：

    • 当leaf node时，判断是否和相等，
      • 如果相等加入结果中
    • 当不是leaf或者结果不相等
      • 判断左子树是否存在
      • 判断右子树是否存在
    • remove last node

  • If you place the backtracking step inside the else block, you risk removing the current node from the path too soon and would mess up the recursive exploration. So, the backtracking step should always be outside the conditional (i.e., after you’ve explored both left and right children), ensuring that the exploration of all paths is done correctly.

  1. Construct Tree from inorder and postorder traversal

  • 基本逻辑：postorder的最后一个元素是用来分隔inorder左右子树的根节点，用此根节点找到inorder的分隔点，再根据inorder的左右子树分隔点，来分隔postorder的左右子树

    • int rootval = postorder[postorder.length - 1];
    • int delimiterIdx => inorder[delimiterIdx] == rootval
    • int[] leftinorder = Arrays.copyOfRange(inorder, 0, delimiterIdx)右括号exclusive
    • int[] rightorder = Arrays.copyOfRange(inorder, delimiterIdx + 1, inorder.length);
    • int leftSize = leftInorder.length();
    • int[] leftpostorder = Arrays.copyOfRange(postorder, 0, leftSize);
    • int[] rightpostorder = Arrays.copyOfRange(postorder, leftSize + 1, postorder.length - 1);
    • recursively find the left and right node to the root

  • Time complexity:

    • O(n) for search the delimiterIdx
    • Since you split inorder into two parts recursively, this results in a total cost proportional to the sum of all array sizes over the recursion: O(n) + O(n-1) + O(n-2) + ... + O(1) = O(n²).
    • Overall O(n^2)

  • Space:

    • Call Stack: The recursion depth is proportional to the height of the tree. In the worst case (a completely unbalanced tree), the height is O(n). In the best case (a balanced tree), the height is O(log n).
    • New Arrays: At each recursive step, new arrays are created for leftInorder, rightInorder, and resizedPostOrder. This creates a space complexity of O(n²) because of the cumulative sizes of all the subarrays created.

  • Optimization
    To improve time and space complexity, avoid creating new subarrays. Instead, use indices to represent the current ranges of the inorder and postorder arrays. This eliminates the need to copy arrays repeatedly.
