代码随想录训练营打卡 Day 20 ｜ 235(LCA for BST), 701(insertion),450 (deletion)

Posted on 2025-01-29 Edited on 2025-02-07 In Medium

今天通过LCA for BST对LCA有了更清晰的理解

 
  1. LCA for BST
    LCA的基本思路是，如果p和q在root的左右不同子树，那么LCA就是root
    如果p和q在同一子树，那么我们搜索root的左子树或者右子树，找到他们分叉的时候，return root。
    终止条件是：如果root == null，那么说明已经到了叶节点，没有找到返回null
    如果root == p或者 root == q说明找到了其中一个节点，返回该节点

如果单纯只是Binary Tree，无法通过节点判断p和q的相对位置，那我们可以使用recursion，根据postorder traversal的顺序，先搜索左子树，再搜索右子树，对当前root进行判断

但如果是BST，我们可以利用它有序的特性，快速判断p，q是否在同侧，减少搜索范围
通过比较p和q和root值之间的大小关系
如果他们同在左侧，那么可以更新root = root.left；右侧同理
如果他们不在同侧，说明root即为LCA

  1. Insert Node into BST
    Iterative
    如果val比root.val更小，那么cur node向左搜索，直到cur node的左节点成为null，在此更新左节点为new TreeNode(val)
    右侧同样
    edge case： 如果root为null，那么直接更新为TreeNode（val）

  2. Delete Node from BST
    迭代法
    Step1:找到该节点
    如果没有找到return root
    Step2:找到之后做判断

  • if 该节点没有子节点，直接删除，注意判断pre是否为null
  • if 该节点有一个子节点，用该子节点替换当前节点，如果是pre的左子就替代左，如果是右就替代右
  • if 该节点有两个节点，那边既可以将删除节点替换成左子树最大值（inorder predecessor)或者右指数最小值(inorder successor)
