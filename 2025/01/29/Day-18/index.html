利用Binary Search Tree有序的特点，缩小搜索的范围

  1. Minimum Absolute Difference in BST
    Naive approach:
    因为BST是有序的，如果用inorder traversal那便会得到一个有序的数组，只需要计算相邻两个数的差值，找到最小即可
    time complexity O(n)(traversal) + O(n)(computation) = O(n)
    space O(n) skewed, O(logN) balanced tree + O(n) traversal list to store node values

Optimized approach:
We don’t need to store all the node values in the inorder traversal, we just need to keep track of the previous node –> pre
WARNING: Java doesn’t update the primitive values in recursion calls, so we have to use class-level variables to ensure that these updates are globally accessible and correctly maintained across the recursion.

Inorder traversal plus keep track of the previous TreeNode pre

  • Recursion: a global TreeNode pre
  • Iteration: inorder traversal
    • push the cur node to the left most node
    • pop the current one
    • traverse the right of the current node
  1. Modes of the BST
    Naive approach:
    Traverse all the nodes in order and count the frequency of the same number

Optimized:
Use the pre node to track the previous node
if pre == cur, freq ++; else freq = 1;
compare the max_freq vs freq, if freq > max_freq update the max_freq and clear the modes -> add the number to the modes array
if max_freq == freq, add the number to the modes array

  1. LCA

Use postorder traversal,
left：search p, q in the left subtree, left = LCA(root.left, p, q)
right：search p, q in the right subtree, right = LCA(root.right, p, q)
root：if left != null && right != null -> LCA is current node
