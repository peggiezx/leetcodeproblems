Day-37 | 动态规划背包问题05

Posted on 2025-03-31 Edited on 2025-04-16
  1. Change Coins
    可以重复选择同一个硬币，那么如果不选择该硬币
    dp[i][j] = dp[i - 1][j]
    如果选择该硬币
    dp[i][j] += dp[i][j - coins[i - 1]]
    注意这里不是dp[i][j] += dp[i - 1][j - coins[i - 1]]因为我们可以重复取统一硬币，因此在同一层反复更新
    amount = 5，coins=【1，2，5】
    初始化如果amount=0，那么无论硬币值多少都可以选择不选因此选法为1
    j 0 1 2 3 4 5

coin = 0，dp=[1, 0, 0, 0, 0, 0]
coin = 1, dp=[1, 1, 1, 1, 1, 1]
coin = 2, dp=[1, 1, 2, 2, 3, 3]
coin = 5, dp=[1, 1, 2, 2, 3, 4]

  1. Combinationa Sum IV
    一、外层遍历“硬币”，内层遍历“金额” —— 得到组合数
    为什么这样就“不考虑顺序”了？
    因为先固定硬币再遍历金额，意味着：

当你在处理 coin=1 时，你把能用 1 凑到的所有方式先记录下来；

然后再处理 coin=2，相当于只是在现有基础上“添加可能使用 2”这种情况。

这样某种组合中“1 和 2”的出现顺序并没有被单独枚举，重复的顺序会被合并成同一条路径。

二、外层遍历“金额”，内层遍历“硬币” —— 得到排列数
为什么这样就“考虑顺序”了？
因为对于每个目标金额 t，你会“立即”去看所有可能放进 t 的硬币，这就相当于“先决定当前总额，再添加任意顺序的硬币”。

某个 t 可能通过先放 1 再放 2，也可以先放 2 再放 1。不同顺序会在不同时刻更新 dp，不会合并成同一个组合。
