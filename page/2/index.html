Day-37 | 动态规划背包问题05

Posted on 2025-03-31 Edited on 2025-04-16
  1. Change Coins
    可以重复选择同一个硬币，那么如果不选择该硬币
    dp[i][j] = dp[i - 1][j]
    如果选择该硬币
    dp[i][j] += dp[i][j - coins[i - 1]]
    注意这里不是dp[i][j] += dp[i - 1][j - coins[i - 1]]因为我们可以重复取统一硬币，因此在同一层反复更新
    amount = 5，coins=【1，2，5】
    初始化如果amount=0，那么无论硬币值多少都可以选择不选因此选法为1
    j 0 1 2 3 4 5

coin = 0，dp=[1, 0, 0, 0, 0, 0]
coin = 1, dp=[1, 1, 1, 1, 1, 1]
coin = 2, dp=[1, 1, 2, 2, 3, 3]
coin = 5, dp=[1, 1, 2, 2, 3, 4]

  1. Combinationa Sum IV
    一、外层遍历“硬币”，内层遍历“金额” —— 得到组合数
    为什么这样就“不考虑顺序”了？
    因为先固定硬币再遍历金额，意味着：

当你在处理 coin=1 时，你把能用 1 凑到的所有方式先记录下来；

然后再处理 coin=2，相当于只是在现有基础上“添加可能使用 2”这种情况。

这样某种组合中“1 和 2”的出现顺序并没有被单独枚举，重复的顺序会被合并成同一条路径。

二、外层遍历“金额”，内层遍历“硬币” —— 得到排列数
为什么这样就“考虑顺序”了？
因为对于每个目标金额 t，你会“立即”去看所有可能放进 t 的硬币，这就相当于“先决定当前总额，再添加任意顺序的硬币”。

某个 t 可能通过先放 1 再放 2，也可以先放 2 再放 1。不同顺序会在不同时刻更新 dp，不会合并成同一个组合。

Day-36 | 动态规划背包问题04

Posted on 2025-02-24 Edited on 2025-04-16 In dp

  1. 将问题转化成，如何将石头分成重量尽量相等的两堆，这样可以保证他们之前的差最小

  2. dp[i][j]代表在前i个石头中，选取若干个石头，使他们的总量不超过目标值j前提下，能达到的最大总和
    j取值是0～target，其中target = sum / 2

  3. 状态转移公式
    如果不选第i个石头：dp[i][j] = dp[i - 1][j]
    如果选第i个石头：dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - cur] + cur);

  4. 初始化
    当 i = 0（没有石头）时，对于任何 j > 0，都无法选取出正数和，所以 dp[0][j] = 0；同时 dp[0][0] = 0。
    对于任意 i，dp[i][0] = 0，因为容量为 0 时只能选取总和为 0 的情况。

  5. 确定遍历顺序
    外层循环：
    从 i = 1 到 n（石头的个数），逐个考虑每个石头。
    内层循环：
    对于每个 i，从 j = 1 到 target（容量）遍历。
    注意：由于状态转移使用的是 dp[i-1][…]，这样可以确保每个石头只使用一次。

  6. Target Sum

  7. 确定如何转化为dp problem。由于所有数字的符号可以为正负，那么假设在原始数字基础上为转换前P数之和与N数之和为所有数字的sum。当选定的N个数转换为负数后P-N = target
    那么P + N = sum && P - N = target
    那么 2*P = sum + target
    P = (sum + target) / 2
    那么问题转换为子集和等于P的数量

  8. 如果P是奇数 或者 P < 0 或者数组之和小于P时都不用继续

  9. 二维背包问题： dp[i][j] = dp[nums.length + 1][p + 1] 意思是在表示前 i 个数中有多少种方式可以使得和为 j。
    e.g. dp[0][0] 在和为0时，如果有0这个选择，那么解题数量为1
    如果不选当前数那么和为j的解决方案数是dp[i - 1][j]
    如果选当前数那么和为j的解法数量是dp[i - 1][j - nums[i - 1]]

  10. Ones and Zeros

  11. 多重维度的dp，m和n：普通的背包问题只有一个容量，单这个题有两个容量，我们用二维的dp在表示在给定的0和1的限额下，最多能放多少字符串

  12. 状态转移：只有在 i >= zeros 且 j >= ones 的情况下，才有可能把它放进去，然后做转移：
    dp[i][j]=max(dp[i][j],dp[i−zeros][j−ones]+1)

  13. 遍历字符串的同时，针对每个字符记录zeros和ones的数量，再针对cur字符来判断是否能够符合m和n的范围，

  14. 倒序遍历以避免重复使用同一个字符串
    如果顺序更新，就会在同一轮反覆利用当前字符串
