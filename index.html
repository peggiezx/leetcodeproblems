什么是单调栈

A monotonic stack is a stack-based data structure that maintains its elements in either increasing or decreasing order as you iterate through an array. It’s commonly used to solve problems like “finding the next greater or smaller element.”

There are two typical patterns:

  • Monotonic Increasing Stack (top is the smallest): When a larger value comes in, you can pop smaller values, and immediately determine the distance.
  • Monotonic Decreasing Stack (top is the largest): When a smaller value appears, you pop all the larger ones off the stack and use the smaller one as the new top, making it easy to compute the position difference.

Posted on 2025-04-04 Edited on 2025-04-07

股票交易

这些题目的核心差异，就是对交易次数、冷却期、手续费等约束的不同。但它们都有一个共同点：你每天只能 买 / 卖 / 休息，且最终的收益最大化。
如前所述，“股票 DP”最常见的模板公式（不考虑冷却、手续费的基本版）是：

dp[i][k][0]=max(dp[i−1][k][0],dp[i−1][k][1]+prices[i])
dp[i][k][1]=max(dp[i−1][k][1],dp[i−1][k−1][0]−prices[i])
解释：
不持股(0)：

  • 或者昨天就不持股，今天不操作；
  • 或者昨天还持股(1)，今天卖掉（所以收益 +prices[i] 并且这笔卖出操作意味着本次交易完成，k 次数不变，因为买+卖才算一次交易，这里已经是卖了，但注意大多数写法里 k 表示“已经完成”的笔数或“还可进行几次交易”——根据你定义不同可能稍微有区别）。

持股(1)：

  • 或者昨天就持股，今天不操作；
  • 或者昨天不持股，今天买入（-prices[i]）。
  • 交易次数 k 要么不变，要么加 1，取决于你在“买”还是在“卖”时算作一次交易。

在“带冷却”或“带手续费”的情况下，公式只是在转移时加一些额外操作（比如 -fee，或者前一天是否冷却）。
