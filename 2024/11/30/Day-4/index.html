打卡第四天！熟悉了更多链表的操作，包括如何交换两个nodes又不使链表断掉，如何用数学的逻辑来简化问题，比如两个nodes之间的距离可以用来解决倒数n的问题；两个链表长度不一样，可以通过相加长度，交换遍历来找到交叉；快慢指针来确定是否链表有环。学到了很多技巧，需要常复习。

  1. Swap Nodes in Pairs
  • initialization: dummy, pre,
  • swap two nodes: first, second, pre
  • move to the next: pre = first
  1. Remove Nth Node From End of List
  • Intuitive sol:
      1. traverse the whole list and find the length
      1. traverse again until index @ (length - n) and do the deletion
    • be careful of the index vs length
  • Optimized: Use two pointers first and second and create a gap of n between two pointer. When first pointer is null, the second pointer points to the node before the node to be removed.
    • e.g. 2th node from the end => we want second points to 3rd from the end, and first points to null => first moves n+1 steps => for (int i = 0; i <= n; i++)
  1. Intersection of Two Linked Lists
  • Intuitive sol:
      1. find the length of listA and listB;
      1. calculate the difference;
      1. move the longer one to the position where shorter one starts;
      1. compare if any node is the same. => we can do nodeA == nodeB to assert.
  • Optimized: if each pointer traverse A + B nodes in total, the intersection node will eventually align
    • while (p1 != p2) => loop exit in 2 scenarios: 1) intersection found; 2) no intersection, pointerA and pointerB will evnetaull reach null, when both are null it stops
    • path equalization: swap to another head when reach the end, the difference in length is neutralized.
  1. Linked List Cycle II
  • KEY1: How to idenitfy the cycle in a linked list
  • SOL1: fast and slow pointer, fast takes 2 steps, slow takes 1. If they meet, there is a cycle
  • KEY2: Once cycle is identified, how can I find where the cycle begins?
  • SOL2: reset fast to the head, and they move together until the meet, that’s where the cycle starts
  • NOTES: how to get the fast and slow start moving => while (fast != null && fast.next != null)
