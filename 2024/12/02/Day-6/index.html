代码随想录训练营打卡 Day 6 | 242. Valid Anagram, 349. Intersection of Two Arrays, 202. Happy Number, 1. Two Sums

Posted on 2024-12-02 Edited on 2024-12-04 In Easy

刷题第6天！需要入魂入脑的总结：当我们遇到了要快速判断一个元素是否出现集合里的时候，就要考虑哈希法。 这次刷题强化了我关于数组也可以用作哈希表的概念， 一般把元素值当成index使用，来判断元素是否存在（0 vs ！0）。如果题目限制数值的大小我们可以用数组，如果没有限制大小就用HashSet或者HashMap。要定义Key和Value到底是什么，重复值是否重要，是要查哪个是否出现过，那那个值就是Key。

  1. Valid Anagram
  • Brute force: sort and compare
  • Optimized: 1) create an array of size 26 (ASCII letters); 2) map the letters to the index ('char' - 'a') and increment the count of chars in s, decrement the count of chars in t; 3) evaluate if there’s a non-zero value in the array
  1. Intersection of Two Arrays
  • Set
  • int[] hash = new int[size of the limit in the prompt]
  1. Happy Number
  • digital manipulation:
    • Extract the last digit: temp = n % 10
    • Perform some operation on the digit (e.g., squaring it, summing it, etc.)
    • Remove the last digit: n = n / 10
  • The key is to find the stopping criteria for the for loop
    • if the number already exists appears again, it will get into the loop
    • use the set to check
  1. Two Sums
  • The tricky part is how to process the duplicated values and define what the Key and what the Value are
  • (k, v) => (nums[i], i) instead of (i, nums[i])
    • easy to check if the remainder of the target - currNum exists
    • we don’t have to worry about storing all the index of the duplicated numbers because there are only two numbers, the one stored in the hashmap is definitely not the current one because we only update the current one after the check of the previous ones
