  1. 4Sum II
    This problem focuses on finding the total number of tuples (a, b, c, d) such that the sum of elements from four integer arrays nums1, nums2, nums3, and nums4 equals zero. Unlike other variants of the “4Sum” problem, it doesn’t require returning the indices of the elements.

The approach simplifies the problem into two 2Sum computations:

  • Compute all possible sums of pairs from nums1 and nums2, and store the frequency of each sum in a hash table.
  • For each possible sum of pairs from nums3 and nums4, check if the negation of that sum exists in the hash table. If it does, the frequency from the hash table contributes to the count of valid tuples.
  1. Ransom Note
  • fixed-size and simple key sets (e.g. letters a to z) => Use array instead of hashmap
  • access speed (constant time O(1))
  • memory efficiency (coniguous memory w overhead of storing keys or managing hash codes)
  • no collision management
